# Epinasty (age-ramped plagiotropism) Implementation Plan

## Overview

Make lateral branches arch toward horizontal over years instead of being
horizontal from node one. This is done by scaling the existing plagiotropism
term in `growth_direction` by an age-dependent ramp `1 - exp(-age/tau)`. Default
off, so existing goldens are bit-for-bit unchanged.

Design spec: `docs/superpowers/specs/2026-05-30-epinasty-design.md`.
Issue: #34. Branch: `issue-34-epinasty-time-dependent-plagiotropic`.

## Current State

- `src/palubicki/sim/tropisms.py` — `growth_direction(*, v_perception,
  current_direction, cfg, is_main_axis, light_gradient=None, axis_order=0)`
  blends perception + orthotropy(UP) + gravitropism(DOWN) + plagiotropism +
  phototropism + inertia. Plagiotropism (lines ~44-73) uses a **constant**
  `w_plagio = cfg.w_plagiotropism_main if is_main_axis else
  cfg.w_plagiotropism_lateral`, scaled by `decay = cfg.axis_decay**axis_order`.
  It projects `current_direction` onto the XY plane (skipping when the direction
  is near-vertical, `|dot(UP)| >= 0.99`) and adds `(w_plagio*decay)*v_plagio` to
  the blend. There is no age input.
- `src/palubicki/config.py` — `TropismConfig` (frozen dataclass) holds the
  weights, each as `field(default=..., metadata={"ui": {"min":..,"max":..,
  "step":..}})`. `photo_direction` and `axis_decay` follow the same style.
- `src/palubicki/sim/simulator.py` (~line 262) — the per-bud growth loop calls
  `growth_direction(...)` with `cfg=cfg.tropism`, `is_main_axis=is_main`,
  `axis_order=bud.axis_order`. The clock time `t` is already in scope here (it is
  passed to `_internode_target(...)` and `_emit_node(...)` in the same loop).
- `src/palubicki/sim/tree.py` — `Internode.birth_time: float = 0.0` (years);
  `Node.parent_internode: Internode | None`; `Bud.parent_node: Node`,
  `Bud.axis_order: int`.
- `src/palubicki/sim/elongation.py` — `shoot_extension` already uses the
  `1 - exp(-x/ref)` saturating form; `update_lengths` reads `cfg.enabled` /
  `cfg.tau_years` and no-ops when disabled. Precedent for the ramp + enable flag.
- Existing tests: `tests/sim/test_tropisms.py` (unit, constructs `TropismConfig`
  and calls `growth_direction` directly), `tests/integration/
  test_plagiotropy_horizontalizes.py` (integration), `tests/golden/` (goldens).

## Desired End State

- `growth_direction` accepts `branch_age_years` and, when epinasty is enabled,
  scales the plagiotropism weight by `1 - exp(-branch_age_years/tau)`.
- The simulator passes `branch_age_years = t - parent_internode.birth_time`.
- Two new `TropismConfig` fields: `epinasty_enabled=False`,
  `epinasty_tau_years=8.0`.
- With epinasty disabled (default), all existing tropism unit tests and goldens
  pass unchanged.
- With epinasty enabled, young branches grow near the parent axis (steep) and
  old branches arch to the configured plagiotropic angle.

## Phase 1: Config fields

### Changes

In `src/palubicki/config.py`, inside `class TropismConfig`, add after the
plagiotropism weights (keep the existing `field(... metadata={"ui": ...})`
style for the float; the bool needs no UI range):

```python
    # Epinasty: ramp the effective plagiotropism weight with branch age,
    # 1 - exp(-age/epinasty_tau_years), so laterals emerge near the parent
    # axis and arch toward horizontal over years (#34). Disabled => ramp is
    # identically 1.0 (bit-identical to the constant-weight behaviour).
    epinasty_enabled: bool = False
    epinasty_tau_years: float = field(
        default=8.0, metadata={"ui": {"min": 0.5, "max": 30.0, "step": 0.5}}
    )
```

## Phase 2: Ramp in `growth_direction`

### Changes

In `src/palubicki/sim/tropisms.py`:

1. Add `import math` (or reuse `np.exp`; `math.exp` on the scalar is fine and
   matches `elongation.py`).
2. Add a keyword-only parameter to `growth_direction`'s signature, after
   `axis_order`:
   ```python
       branch_age_years: float = 0.0,
   ```
3. Where `w_plagio` is selected, scale it by the ramp:
   ```python
   w_plagio = cfg.w_plagiotropism_main if is_main_axis else cfg.w_plagiotropism_lateral
   if cfg.epinasty_enabled and cfg.epinasty_tau_years > 0.0:
       ramp = 1.0 - math.exp(-max(0.0, branch_age_years) / cfg.epinasty_tau_years)
       w_plagio = w_plagio * ramp
   ```
   Leave the rest of the plagiotropism block (near-vertical skip, projection,
   renormalisation) and the final `blend` unchanged. Because the existing block
   already guards `if w_plagio > 0.0`, a ramp of 0 at age 0 cleanly disables the
   term (the lateral then follows inertia/orthotropy/perception — i.e. stays near
   the parent axis).

### Checkpoint

- [ ] Run: `.venv/bin/pytest tests/sim/test_tropisms.py -q`
- [ ] Expect: all existing tests pass unchanged (epinasty defaults off, so
      `w_plagio` is unmodified).

## Phase 3: Unit tests for the ramp

### Changes

Add to `tests/sim/test_tropisms.py`:

```python
def test_epinasty_disabled_matches_constant_weight():
    """Default (disabled) epinasty leaves plagiotropism at full strength
    regardless of branch_age_years."""
    cfg = TropismConfig(
        w_perception=0.0, w_orthotropy_main=0.0, w_orthotropy_lateral=0.0,
        w_gravitropism_main=0.0, w_gravitropism_lateral=0.0,
        w_plagiotropism_main=0.0, w_plagiotropism_lateral=1.0,
        w_phototropism=0.0, w_direction_inertia=0.0,
    )
    cur = np.array([1.0, 1.0, 0.0]) / np.sqrt(2.0)
    base = growth_direction(v_perception=np.zeros(3), current_direction=cur,
                            cfg=cfg, is_main_axis=False)
    for age in (0.0, 5.0, 50.0):
        d = growth_direction(v_perception=np.zeros(3), current_direction=cur,
                             cfg=cfg, is_main_axis=False, branch_age_years=age)
        np.testing.assert_allclose(d, base, atol=1e-12)


def test_epinasty_young_branch_ignores_plagiotropism():
    """With epinasty on and age 0, the lateral keeps its current direction
    (ramp=0 disables the horizontal pull; only inertia remains)."""
    cfg = TropismConfig(
        w_perception=0.0, w_orthotropy_main=0.0, w_orthotropy_lateral=0.0,
        w_gravitropism_main=0.0, w_gravitropism_lateral=0.0,
        w_plagiotropism_main=0.0, w_plagiotropism_lateral=1.0,
        w_phototropism=0.0, w_direction_inertia=1.0,
        epinasty_enabled=True, epinasty_tau_years=8.0,
    )
    cur = np.array([1.0, 1.0, 0.0]) / np.sqrt(2.0)
    d = growth_direction(v_perception=np.zeros(3), current_direction=cur,
                         cfg=cfg, is_main_axis=False, branch_age_years=0.0)
    np.testing.assert_allclose(d, cur, atol=1e-7)


def test_epinasty_old_branch_recovers_full_plagiotropism():
    """With epinasty on and age >> tau, the horizontal pull approaches the
    full-strength (disabled) result."""
    common = dict(
        w_perception=0.0, w_orthotropy_main=0.0, w_orthotropy_lateral=0.0,
        w_gravitropism_main=0.0, w_gravitropism_lateral=0.0,
        w_plagiotropism_main=0.0, w_plagiotropism_lateral=1.0,
        w_phototropism=0.0, w_direction_inertia=0.0,
    )
    cur = np.array([1.0, 1.0, 0.0]) / np.sqrt(2.0)
    full = growth_direction(v_perception=np.zeros(3), current_direction=cur,
                            cfg=TropismConfig(**common), is_main_axis=False)
    cfg_on = TropismConfig(**common, epinasty_enabled=True, epinasty_tau_years=8.0)
    d_old = growth_direction(v_perception=np.zeros(3), current_direction=cur,
                             cfg=cfg_on, is_main_axis=False, branch_age_years=80.0)
    np.testing.assert_allclose(d_old, full, atol=1e-3)
    # horizontal: no vertical component
    assert abs(float(np.dot(d_old, [0.0, 1.0, 0.0]))) < 1e-3


def test_epinasty_monotone_in_age():
    """The horizontal component of the result increases monotonically with age."""
    cfg = TropismConfig(
        w_perception=0.0, w_orthotropy_main=0.0, w_orthotropy_lateral=0.0,
        w_gravitropism_main=0.0, w_gravitropism_lateral=0.0,
        w_plagiotropism_main=0.0, w_plagiotropism_lateral=1.0,
        w_phototropism=0.0, w_direction_inertia=1.0,
        epinasty_enabled=True, epinasty_tau_years=8.0,
    )
    cur = np.array([1.0, 1.0, 0.0]) / np.sqrt(2.0)
    horiz = []
    for age in (0.0, 4.0, 8.0, 16.0, 32.0):
        d = growth_direction(v_perception=np.zeros(3), current_direction=cur,
                             cfg=cfg, is_main_axis=False, branch_age_years=age)
        horiz.append(d[0])  # x grows as the vector flattens toward horizontal
    assert all(b >= a - 1e-9 for a, b in zip(horiz, horiz[1:]))
```

        parent_iod = bud.parent_node.parent_internode

## Rollback / safety notes

- The entire feature is gated by `epinasty_enabled` (default `False`). If
  anything misbehaves, the default path is identical to pre-change behaviour.
- Node positions are fixed at creation (`elongation.py`), so this only affects
  newly-emitted growth — there is no retro-active geometry mutation to worry
  about.
- Deferred (separate issues, not in this plan): epinastic droop (sub-horizontal
  set-point), active re-straightening via reaction wood, per-axis-order tau.
